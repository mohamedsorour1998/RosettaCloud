Container Startup Sequence

What happens inside the lab pod after creation — readiness probe passes early so users aren't blocked

Phase 1 — ~3-6s
Pod Ready
code-server (port 8080) + Caddy (port 80 reverse proxy) start. Readiness probe HTTP GET / :80 succeeds → pod marked Ready. User can open terminal.
Phase 2 — ~5-15s
Docker Ready
dockerd starts in background. Waits for docker info to succeed. No user impact — happens concurrently.
Phase 3 — ~10-30s
Image Loaded
docker load -i /kind-node.tar loads the 650 MB+ Kind node image. Background operation — terminal is already usable.
Phase 4 — ~30-60s
Cluster Ready
kind create cluster provisions a full Kubernetes cluster inside the pod. CPU-intensive. kubectl becomes available.

Chatbot Request Flow

End-to-end path from Angular to AgentCore Runtime and back

1
Angular ChatbotComponent sends POST /chat
Body: {{ '{' }}session_id, message, user_id, module_uuid, lesson_uuid, type{{ '}' }}
session_id format: "session-{{ '{' }}uuid{{ '}' }}-{{ '{' }}timestamp{{ '}' }}" — must be ≥33 chars (AgentCore requirement)
2
FastAPI /chat endpoint loads history from _chat_histories[session_id]
Max 40 messages, 4h TTL, 500 sessions in-process — single pod makes this fully consistent
3
boto3 calls invoke_agent_runtime on the AgentCore Runtime
Synchronous call via bedrock-agentcore service client
4
AgentCore routes to the appropriate agent based on message intent
Tutor Grader Planner
5
Agent calls tools via AgentCore Gateway → Lambda agent_tools
MCP protocol: education-tools___get-question-details → agent normalizes name → Gateway strips prefix → Lambda dispatches
6
Response {{ '{' }}response, agent, session_id{{ '}' }} returned to FastAPI
FastAPI saves updated history to _chat_histories
7
Angular renders markdown response; agent field identifies which agent responded

How shell script questions become searchable knowledge for the Tutor agent

Feedback Flow
Feedback uses the same AgentCore Runtime — no separate service or pub/sub.

When the user requests end-of-lab feedback, Angular calls ChatbotService.sendFeedbackRequest() which sends POST /chat with type: "feedback", the questions array, and user progress. The Planner agent generates educational feedback and returns it as a normal chat response.
